# Plugin requirements and submission

## Requirements

The following requirements are here to provide a secure and unified experience for Ledger devices users.

To make sure the review process of your Embedded Plugin goes smoothly and quickly, we recommend you read the requirements before starting to code.

| Category        | Requirement                                                                                                     |
|-----------------|-----------------------------------------------------------------------------------------------------------------|
|  General        | Communicate to Ledger the DApp name, the smart contract address (if applicable), and an email address.          |
| Coding          | Go through the [Plugin Code walkthrough](./code-walkthrough) to create your plugin.   |
| Design          | Follow the [Device app design requirements](../device-app/coding-guidelines).               |
| Documentation   | Follow the [Device app documentation requirements](../device-app/deliverables/documentation-requirements). |
| Support         | Follow the [Device app support requirements](../device-app/deliverables/support-requirements).             |
| Security Audits | Follow the [Device app security audits requirements](../device-app/submission/security-audit).          |


## Submission

Once your plugin is ready, please [fill in this form](https://ledger.typeform.com/to/YZZCHEB8).
